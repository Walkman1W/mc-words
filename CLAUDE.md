## Project Overview

Minecraft Word Cards — a browser-based English vocabulary learning game themed around Minecraft. Players spell words by dragging letters, unlock cards progressively, and compete on time-based leaderboards.

## Commands

```bash
# Start dev server (serves on http://localhost:8080)
node server.js

# Regenerate manifest.json from card image filenames
node generate-manifest.js
```

No build step, no package.json, no bundler. Pure vanilla HTML/CSS/JS with ES modules loaded via `<script type="module">`.

## Architecture

**Two pages:**
- `index.html` — Landing/login page (username entry with history dropdown)
- `gallery.html` — Main app (card gallery, spelling game modal, leaderboard modal)

**JS modules (all in `js/`):**
- `auth.js` — Multi-user management via localStorage (`mc-users`, `mc-current-user`)
- `landing-app.js` — Login page logic with user history dropdown
- `gallery-app.js` — App bootstrap: auth guard, loads data, inits all modules
- `gallery.js` — Tab navigation, card grid rendering, favorites tab, leaderboard UI
- `categories.js` — Category definitions (static list of `{id, label}`)
- `cardData.js` — Fetches `manifest.json` at runtime, provides card lookup
- `game.js` — Spelling game: drag-and-drop letters, auto-submit, timeout, result display with favorites
- `progress.js` — Per-user progress + favorites via localStorage (`mc-progress-{user}`, `mc-favorites-{user}`)
- `timer.js` — Per-card timing, category totals, leaderboard with seeded NPC data

**CSS (all in `css/`):** `main.css` (reset + shared variables/buttons), `landing.css`, `gallery.css`, `game.css`, `timer.css`

**Data flow:** `server.js` dynamically generates manifest from filesystem on `/assets/images/cards/manifest.json` requests. The static `manifest.json` file (generated by `generate-manifest.js`) is the fallback for non-server environments.

## Card Image Convention

Images must follow the pattern `{3-digit-id}-{Word Name}.png` (e.g., `012-Diamond Helmet.png`). Only files matching this pattern appear in the manifest. Categories are directories like `01-block/`, `02-tool/`, etc.

## Key Data Structures (localStorage)

- `mc-users` — JSON array of all registered usernames
- `mc-current-user` — Currently logged-in username string
- `mc-progress-{username}` — `{ categoryId: [unlockedCardIndices] }`
- `mc-favorites-{username}` — `{ categoryId: [cardIds] }`
- `mc-timer-{username}` — Per-card timing records

## Configuration

`config.json` controls gameplay:
- `levelLock.showWordOnLocked` / `showThumbnailOnLocked` — visibility of locked cards
- `timer.secondsPerLetter` — time limit = word length × this value

## CSS Variables (defined in main.css)

Key theme colors: `--mc-green`, `--mc-gold`, `--mc-brown`, `--mc-dark-brown`, `--mc-stone`, `--mc-dark`, `--mc-darker`, `--mc-light`. Font: `--font-mc` (Press Start 2P).
